# Product Theory

## Why this deserves to exist

Modern structural biology is rich in data but poor in interfaces that preserve both scale and meaning. Existing tools often force users to choose between an overwhelming data catalogue, a specialist desktop application, or a text assistant detached from the object being discussed.

Helicase treats the molecular object, the evidence about it, and the reasoning around it as one navigable system.

## Core belief

Reasoning becomes more useful when it can act on the world it is describing. In Atlas, GPT-5.6 is not a chat window beside a viewer. It is a constrained scientific guide whose typed actions focus residues, changes representation, exposes uncertainty, compares structures, and launches a clearly labelled design playback.

## The emotional arc

1. **Wonder:** a protein universe appears as a living field.
2. **Orientation:** a selected star resolves into a real structure.
3. **Trust:** confidence and provenance are made legible.
4. **Agency:** the user asks a meaningful scientific question.
5. **Creation:** a design hypothesis appears through an explicit, honest pipeline.

## The unforgettable moment

The system should move from a spoken or typed molecular intention to a visible, causally connected scene change. The user should understand what GPT decided, what action it took, what evidence supports it, and what remains uncertain.

## Product test

If a feature cannot improve discovery, understanding, or causally visible creation, it does not belong in Atlas.
