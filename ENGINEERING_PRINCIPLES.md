# Engineering Principles

- Keep domain logic independent from rendering libraries.
- Make invalid states difficult to represent through strict TypeScript contracts.
- Prefer deterministic fixtures and reproducible tours for demo-critical behavior.
- Keep asynchronous work cancellable and observable.
- Test command semantics separately from camera and shader effects.
- Keep scientific provenance adjacent to the value it qualifies.
- Never hide a fallback behind language that implies live generation or validation.
- Add complexity only when it creates user-visible capability or protects scientific integrity.
- Document decisions when a shortcut changes product meaning.
- Optimize performance from the scene graph and data shape outward; do not pre-emptively add infrastructure.
- Keep external providers replaceable through adapters.
- Treat the README and architecture docs as maintained product surfaces.
