# Build Plan

## Phase 0 — Foundation

Create the application shell, strict TypeScript configuration, domain contracts, fixture loader, and a minimal SceneController. Prove a single Three.js scene and a single Mol* structure can coexist.

## Phase 1 — The dive

Render curated protein stars, support hover and selection, and animate a deterministic map-to-structure transition. Load one real structure with confidence coloring.

## Phase 2 — Design reveal

Implement one DesignScene that plays a real precomputed trajectory from noise through fold and docking to score disclosure. Make provenance and predicted-not-validated status visible.

## Phase 3 — Copilot control

Add the server-side GPT-5.6 call, compact context assembly, streaming, schema validation, and the first two tools: `color_by(trusted_core)` and `design_binder`.

## Phase 4 — Scientific depth

Add residue focus, domain highlighting, representation changes, comparison, citations, and confidence X-ray behavior.

## Phase 5 — Experience polish

Refine choreography, typography, lighting, loading, interruption, guided tours, responsive behavior, reduced motion, and performance.

## De-risk order

Prove renderer coexistence first, then the dive, then one complete design reveal, then GPT orchestration. Do not build broad APIs or infrastructure before the headline path is visually real.

## Definition of done

The product opens without login, communicates what it is within seconds, supports a complete map-to-structure-to-question-to-design flow, makes uncertainty visible, cites its scientific inputs, survives unavailable services through labelled fallbacks, and can be demonstrated without manual repair.
