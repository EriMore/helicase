# Interaction Principles

## Continuous space

Map, structure, confidence view, and design playback are modes of one instrument, not separate pages. Transitions should preserve orientation and explain what changed.

## Direct manipulation and delegation

Mouse, keyboard, tours, and GPT all issue the same semantic commands. No interaction path should receive privileged or hidden behavior.

## Feedback

Every command has an immediate acknowledgement: camera motion, loading state, streamed reasoning, a focused residue, or a visible error. Long operations reveal progress and provenance.

## Scientific honesty

The interface distinguishes observed structures, predicted structures, generated candidates, and hypothetical interpretations. A polished animation must never imply experimental certainty.

## Motion grammar

Reuse a small vocabulary: `dive`, `surface`, `focus`, `morph`, `reveal-confidence`, `denoise`, and `wobble`. Each transition has a semantic purpose and a reduced-motion alternative.

## Recovery

Users can interrupt a transition, return to the map, dismiss the copilot, and retry a failed fetch without losing the selected context.
