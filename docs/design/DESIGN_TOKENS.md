# Design Tokens

These tokens define the vocabulary for implementation. Exact values may be tuned after visual testing, but semantic names should remain stable.

## Color tokens

```text
--space-950       deep spatial background
--space-900       elevated spatial layer
--ink-primary     primary readable text
--ink-secondary   metadata and supporting text
--signal-trust    high-confidence blue/cyan
--signal-doubt    low-confidence amber/orange
--signal-create   generation violet
--signal-alert    error red
```

## Motion tokens

```text
--motion-ack      immediate intent acknowledgement
--motion-focus    residue/domain focus
--motion-dive     map-to-structure travel
--motion-reveal   confidence or design resolution
--motion-settle   final readable state
```

## Spatial tokens

```text
--space-unit      base spacing unit
--scene-gutter    minimum safe scene margin
--rail-width      identity/orientation rail width
--copilot-width   contextual reasoning surface width
--focus-radius    default focus halo size
```

## Type tokens

```text
--type-display
--type-interface
--type-evidence
--type-meta
```

Tokens must encode meaning, not individual component preferences.
