# Interaction System

## Navigation philosophy

Navigation is spatial, not page-based. The user moves through nested scales of biology. A persistent orientation cue identifies current scale and the route back to the map.

## Selection

Hover creates a small halo and a concise label. Selection creates a committed light path, a camera destination, and a short identity statement. Never make the user infer whether a star is selectable.

## Command surfaces

Use direct manipulation for looking and a conversational surface for asking. The copilot is contextual: it appears as a calm instrument when needed and recedes when the object should dominate.

## GPT actions

The copilot's tool calls are visible as actions, not hidden automation. A focus command visibly moves the camera; `color_by(trusted_core)` visibly invokes the X-ray; `design_binder` visibly begins a named pipeline.

## Progressive disclosure

Immediate: object identity, current scene, action affordance.

On demand: domains, residue evidence, source records, comparisons, model explanation.

Always available but quiet: confidence legend, provenance, prediction disclaimer, return-to-map.

## Feedback and recovery

Every command has acknowledgement, progress, and a recoverable outcome. Errors explain what failed and preserve the last useful scene. The user can interrupt any camera or reveal transition.

## Keyboard and accessibility

All critical commands have keyboard equivalents. Focus is visible. Color is paired with labels or texture. Reduced motion preserves semantic order without requiring animation.
